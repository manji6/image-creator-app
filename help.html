使い方ガイド

Image Batch Studioの基本的な使い方を説明します。

Image Batch Studioとは

複数のプロンプトから一括で画像を生成できる、ローカルファースト設計のWebアプリケーションです。 すべてのデータ（設定、カード、APIキー）はブラウザ内のIndexedDBにのみ保存され、サーバー側には一切送信されません。

基本的な使い方

1. プロバイダーを選択

fal.ai、Google AI Studio、Adobe Firefly の3つから選択できます。 それぞれのAPIキーを入力してください。

2. モデルを選択

「一覧を更新」ボタンでモデル一覧を取得できます。手動入力も可能です。

3. プロンプトを入力

共通プロンプトと行ごとのプロンプトを入力します。1行 = 1枚のカードが生成されます。

4. カードを作成して生成

「カードを作成」ボタンでプロンプトカードを生成し、「カードを全て生成」で一括生成を開始します。

5. 結果をダウンロード

個別ダウンロードまたは全カード一括ダウンロード（ZIP形式）が可能です。

モード切替

Lightモード

共通プロンプト + 各行プロンプトを単純に連結します。シンプルで分かりやすい入力方法です。

例: 共通「かわいい猫」+ 行「青い目」 → 「かわいい猫 青い目」

Advancedモード

共通プロンプト内で {{item}} を使ってテンプレート展開できます。 ${item} や {$item} も互換性があります。

例: 共通「{{item}}の猫」+ 行「青い」 → 「青いの猫」

プロバイダー別の設定

fal.ai

  • API Keyを入力
  • モデル一覧から選択または手動入力
  • 一部モデルは参照画像の入力が必要（自動判定）

Google AI Studio

  • Google API Keyを入力
  • Geminiモデルから選択
  • デフォルト: gemini-2.5-flash-image-preview

Adobe Firefly

現在はDirectモードのみ対応しています:

  • Client ID + Access Tokenを直接入力
  • Proxyモードは今後準備予定

詳しくは Fireflyガイド をご覧ください。

よくある質問

Q. APIキーはどこに保存されますか？

ブラウザのIndexedDBにのみ保存されます。サーバーには送信されません。

Q. Firefly Access Tokenは保存されますか？

セッション中のみメモリに保持され、「設定を保存」しても永続化されません。

Q. 生成した画像はどこに保存されますか？

カード情報（画像URLを含む）がIndexedDBに保存されます。実際の画像ファイルは各プロバイダーのサーバーにあります。

Q. 並行生成数は変更できますか？

デフォルトは2に設定されています。
